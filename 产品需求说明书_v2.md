# 产品需求说明书 v2.0

> 求职助手 · 产品需求文档

---

## 1. 产品概述

### 1.1 产品定位

AI 驱动的简历分析 + 职位浏览工具，帮助用户快速了解自身与目标岗位的匹配度，并获得针对性的简历优化建议和面试准备指导。

### 1.2 目标用户

- 正在求职的互联网/新能源汽车/AI 行业从业者
- 希望通过 AI 分析了解自身竞争力的求职者
- 需要批量浏览猎聘职位并快速筛选的用户

### 1.3 核心价值

- **省时间**：不用逐个点开职位看 JD，AI 秒级匹配
- **有洞见**：量化简历与岗位的匹配度，发现盲区
- **可落地**：给出具体改进建议和真实面试题

## 3. 功能需求详情

### 3.1 职位列表页（index.html / jobs.html）

#### 3.1.1 职位数据展示

- 从同目录 `liepin_job_list.csv` 读取数据
- 显示字段：公司名、岗位名（缩略）、城市、薪资、匹配度分数
- 列表按匹配度从高到低排序
- 匹配度 >= 80：高（绿色）；60-79：中（黄色）；< 60：低（红色）

#### 3.1.2 交互行为

| 操作 | 行为 |
|------|------|
| 点击职位卡片 | 跳转 `analyze.html?job=<编码后的岗位JSON>` |
| 页面加载 | 显示"加载中…"占位，数据就绪后渲染列表 |

#### 3.1.3 页面统计信息

- 共 X 个职位（动态读取 CSV 行数）
- 城市：北京 / 上海 / 杭州
- 来源：猎聘

---

### 3.2 简历分析页（analyze.html）

#### 3.2.1 文件上传

- 支持格式：`application/pdf`、`application/vnd.openxmlformats-officedocument.wordprocessingml.document`（.docx）、`application/msword`（.doc）
- 最大文件大小：5MB（建议值）
- 上传方式：点击区域上传 或 拖拽上传
- 文件解析：PDF.js 解析 PDF，JSZip + Mammoth.js 解析 DOCX

#### 3.2.2 岗位信息展示

- 从 URL 参数 `?job=xxx` 解析岗位数据
- 展示：公司名、岗位名、薪资、城市、岗位描述（摘要）
- 无参数时显示手动输入表单

#### 3.2.3 手动补充字段（可选）

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | 简历未提取到姓名时手动补充 |
| 目标岗位 | 文本 | 可覆盖 URL 参数中的岗位名 |

#### 3.2.4 AI 分析流程

**Step 1**：用户点击"开始分析"，进入加载状态

**Step 2**：向 Coze API 发起聊天请求（POST /v3/chat）

**Step 3**：轮询获取结果（GET /v3/chat/retrieve + GET /v3/chat/message/list），轮询间隔 2 秒，超时 60 秒

**Step 4**：解析 AI 返回结果，渲染分析报告

#### 3.2.5 AI 分析报告内容

| 模块 | 内容 |
|------|------|
| 综合匹配分 | 0-100 分，大字展示 |
| 评级文字 | 优秀 / 良好 / 一般 / 较差 |
| 六项维度评分 | 技能匹配、经验匹配、学历匹配、行业匹配、薪资匹配、整体竞争力 |
| 维度柱状图 | 每项维度的分值可视化 |
| 匹配亮点 | AI 识别出的简历优势（绿色标签） |
| 差距分析 | AI 识别出的弱项（红色标签） |
| 改进建议 | 3-5 条具体可操作的建议 |
| 面试题推荐 | 3-5 道目标岗位常见面试题 |

## 4. 数据结构

### 4.1 职位 CSV 格式

文件：`liepin_job_list.csv`，UTF-8 with BOM

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 行业 | 所属行业 | 汽车/新能源/人工智能 |
| 城市 | 工作城市 | 北京 |
| 岗位角色 | 岗位类别 | 产品经理/项目经理/技术经理 |
| 关键词 | 匹配关键词 | 人工智能/智能驾驶 |
| 职位名 | 具体职位名称 | 交付总监 |
| 薪资 | 薪资范围 | 30-40k |
| 公司名 | 公司全称 | 美团 |
| 匹配行业关键词 | 内部匹配标记 | 车联网 |
| job_id | 猎聘职位 ID | 1944774085 |
| 岗位描述 | JD 全文 | 岗位职责... |
| URL | 猎聘移动端链接 | https://m.liepin.com/job/xxx |
| 匹配度 | 0-100 分 | 85 |
| Action | 备用字段 | 空 |

### 4.2 岗位数据对象（JS）

```json
{
  "company": "公司名",
  "title": "职位名",
  "desc": "岗位描述（截断至200字）",
  "fullDesc": "岗位描述（完整）",
  "score": "匹配度分数",
  "rawScore": "原始分数",
  "salary": "薪资",
  "city": "城市",
  "url": "猎聘链接"
}
```

## 5. 非功能性需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 首屏加载时间 | < 2 秒（文件合计 < 300KB） |
| AI 分析等待 | 显示加载动画，60 秒内返回结果 |
| 列表渲染 | 101 条数据，< 500ms 完成 |

### 5.2 兼容性

- 浏览器：Chrome 90+、Safari 14+、Firefox 88+、Edge 90+
- 移动端：iOS Safari 14+、Android Chrome 90+
- 不支持 IE

### 5.3 可访问性

- 上传按钮、卡片链接均有文字说明
- 色彩对比度符合 WCAG AA 标准

## 6. 已知限制

| 限制 | 说明 |
|------|------|
| 简历解析 | DOCX 依赖 Mammoth.js，复杂格式可能丢失部分内容 |
| API 稳定性 | Coze API 为第三方服务，需确保 Token 有效 |
| Token 安全 | PAT Token 嵌在 JS 中，不适合公开分享给大量未知用户 |
