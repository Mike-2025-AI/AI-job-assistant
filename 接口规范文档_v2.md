# 接口规范文档 v2.0

> 求职助手 · 外部接口文档

---

## 1. 接口概述

本项目调用两个外部接口：

| 接口 | 用途 | 调用方 |
|------|------|--------|
| Coze API v3 | AI 简历+岗位匹配分析 | analyze.html |
| 猎聘（数据源） | 职位原始数据来源 | 离线 CSV（手动采集） |

---

## 2. Coze API 规范

### 2.1 基础信息

| 字段 | 值 |
|------|---|
| 域名 | `https://api.coze.cn` |
| 版本 | v3 |
| 认证方式 | Bearer Token（PAT） |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |

### 2.2 认证

```http
Authorization: Bearer {PAT_TOKEN}
Content-Type: application/json
```

> ⚠️ **Token 安全说明**：PAT（Personal Access Token）嵌在前端 JS 中，仅能调用授权给该 Token 的 Bot，无法访问账号其他资源。但任何人可从公开页面源码中获取，请勿用于公开网站。

### 2.3 接口一：发起聊天（核心）

**请求**
```http
POST https://api.coze.cn/v3/chat
```

**请求体**
```json
{
  "bot_id": "7500000000000000000",
  "user_id": "job-user-1742400000000",
  "stream": false,
  "auto_save_history": false,
  "additional_messages": [
    {
      "role": "user",
      "content": "请分析简历与岗位的匹配度。\n\n简历内容：\n{用户简历全文}\n\n岗位信息：\n{公司名} · {职位名}\n{薪资范围} · {城市}\n\n岗位职责：\n{岗位描述全文}",
      "content_type": "text"
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bot_id | string | ✅ | Coze Bot 的唯一 ID |
| user_id | string | ✅ | 用户标识，建议带时间戳避免重复 |
| stream | boolean | ✅ | 是否启用流式，返回 `false` |
| auto_save_history | boolean | ✅ | 是否自动保存对话历史，`false` |
| additional_messages | array | ✅ | 用户消息数组 |
| additional_messages[].role | string | ✅ | 固定值 `"user"` |
| additional_messages[].content | string | ✅ | 完整提示词 |
| additional_messages[].content_type | string | ✅ | 固定值 `"text"` |

**响应**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "chat_id": "7500000000000000001",
    "conversation_id": "7500000000000000002"
  }
}
```

**成功响应码**：`code === 0`

### 2.4 接口二：查询聊天状态

**请求**
```http
GET https://api.coze.cn/v3/chat/retrieve?chat_id={chat_id}&conversation_id={conversation_id}
```

**响应**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "id": "7500000000000000001",
    "conversation_id": "7500000000000000002",
    "status": "completed",
    "created_at": 1742400000,
    "completed_at": 1742400030
  }
}
```

| status 值 | 含义 |
|-----------|------|
| `in_progress` | 进行中，需继续轮询 |
| `completed` | 已完成，可获取结果 |
| `failed` | 失败，终止轮询 |

---

### 2.5 接口三：获取消息列表

**请求**
```http
GET https://api.coze.cn/v3/chat/message/list?chat_id={chat_id}&conversation_id={conversation_id}
```

**响应**
```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "id": "7500000000000000003",
        "role": "user",
        "type": "question",
        "content": "简历...",
        "content_type": "text"
      },
      {
        "id": "7500000000000000004",
        "role": "assistant",
        "type": "answer",
        "content": "## 综合匹配度：85\n\n## 六维评分\n...",
        "content_type": "text"
      }
    ]
  }
}
```

> 取 `type === "answer"` 且 `role === "assistant"` 的消息，即为 AI 分析结果。

## 3. Coze Bot 提示词设计

> 以下为 Coze Bot 的系统提示词参考，用于定义 AI 分析的行为和输出格式。

```
你是一位专业的简历评估专家。请根据以下简历和岗位信息，从六个维度进行深入分析：

## 评分维度（每项0-100分）
1. 技能匹配度：该候选人的技能与岗位要求的匹配程度
2. 经验匹配度：工作年限和经历与岗位的契合程度
3. 学历匹配度：学历背景与岗位要求的匹配程度
4. 行业匹配度：候选人所处行业与目标公司的契合程度
5. 薪资匹配度：候选人期望薪资与岗位薪资范围的匹配程度
6. 整体竞争力：该候选人在同类候选人中的相对竞争力

## 输出格式（严格按照以下格式，不要偏离）
## 综合匹配度：X分
## 评级：[优秀/良好/一般/较差]

## 六维评分
技能匹配：X分
经验匹配：X分
学历匹配：X分
行业匹配：X分
薪资匹配：X分
整体竞争力：X分

## 匹配亮点
（列出3-5个简历中与岗位高度匹配的亮点，使用「•」列表）

## 差距分析
（列出3-5个简历中与岗位存在差距的地方，使用「•」列表）

## 改进建议
（针对差距给出3-5条具体可操作的改进建议，使用「•」列表）

（推荐3-5道该岗位常见的面试题，使用「•」列表）
```

## 4. 错误处理

### 4.1 HTTP 错误码

| 状态码 | 含义 | 处理方式 |
|--------|------|----------|
| 200 | 成功 | 解析响应体 |
| 400 | 请求参数错误 | 检查请求体格式 |
| 401 | Token 无效或过期 | 刷新 Coze PAT |
| 403 | 无权限调用该 Bot | 检查 Bot ID 和 Token 授权 |
| 429 | 请求过于频繁 | 等待后重试，建议间隔 2 秒 |
| 500 | Coze 服务端错误 | 等待后重试 |

### 4.2 业务层错误

```javascript
if (!initResp.ok) {
  const errText = await initResp.text();
  throw new Error(`API请求失败(${initResp.status}): ${errText}`);
}
if (!chat_id) throw new Error('API未返回chat_id');
// 轮询超时时
throw new Error('分析超时，请重试');
```

## 5. 数据透传参数（内部）

用于从 `jobs.html` 传递职位数据到 `analyze.html`，不属于外部 API。

| 参数名 | 位置 | 格式 | 说明 |
|--------|------|------|------|
| `job` | URL Query String | base64 编码的 JSON | 完整职位对象 |

解码方式：
```javascript
const jobData = JSON.parse(
  decodeURIComponent(
    atob(new URLSearchParams(location.search).get('job'))
  )
);
```

## 6. 日调用限制

| 限制项 | 数值 | 来源 |
|--------|------|------|
| Coze PAT 日调用上限 | 100 次/天（免费版参考值） | Coze 平台策略 |
| 单次分析耗时 | 10-60 秒 | 网络延迟 |
| 建议轮询间隔 | 2 秒/次 | 前端实现 |

> ⚠️ 以上限制值为参考值，实际限制可能因 Coze 平台政策调整而变化，建议在 Coze 控制台确认当前配额。
